# Repair Bot — Telegram-бот для приёма заявок на ремонт

## Быстрый старт

### 1. Клонировать и войти в папку
```bash
cd repair-bot
```

### 2. Создать виртуальное окружение
```bash
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
```

### 3. Установить зависимости
```bash
pip install -r requirements.txt
```

### 4. Настроить переменные окружения
```bash
cp env.example .env
# Отредактируйте .env: вставьте BOT_TOKEN и MASTER_CHAT_ID
```

**Как получить BOT_TOKEN**: создайте бота через @BotFather в Telegram.

**Как получить MASTER_CHAT_ID** — два варианта:

1. **Личный чат (простой, для одного клиента в момент времени)** — отправьте сообщение боту [@userinfobot](https://t.me/userinfobot), он покажет ваш `Id`. Заявки будут приходить вам в личку с привязкой через reply_to.

2. **Форум-супергруппа (рекомендуется при >1 заявке параллельно)** — каждая заявка получит отдельный топик-окно:
   - Создайте новую группу в Telegram → меню → «Создать группу»
   - В настройках группы переключите в супергруппу (или сделайте публичной/приватной)
   - Включите тумблер **«Темы»** (Topics)
   - Добавьте бота как **админа** с правом **«Управление темами»**
   - Получите ID группы через [@RawDataBot](https://t.me/RawDataBot) — это число со знаком минус, начинается с `-100…`
   - Подставьте этот ID в `MASTER_CHAT_ID`

   В форум-режиме бот автоматически создаёт топик на каждую заявку. Вся переписка по заявке (уведомление, фото, голос, реплики клиента, ответы мастера) живёт в этом топике. Ответ клиенту = просто написать в нужный топик.

### 5. Запустить бота
```bash
python -m bot.main
```

```
├── requirements.txt

```

## Сценарий использования

**Клиент:**
1. `/start` — выбирает устройство
2. Описывает проблему (текст или голос)
3. Указывает телефон
4. Подтверждает заявку

**Мастер:**
1. Получает уведомление с деталями заявки
2. Отвечает командой `/reply_<номер_заявки>`
3. Вводит текст ответа — бот пересылает клиенту

## База данных

Файл `orders.db` создаётся автоматически при первом запуске.

Просмотр заявок:
```bash
sqlite3 orders.db "SELECT id, device_type, phone, status, created_at FROM orders;"
```

## Переменные окружения

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен бота от @BotFather |
| `MASTER_CHAT_ID` | ID чата мастера (личка или группа) |
| `DATABASE_PATH` | Путь к SQLite-файлу (по умолчанию `orders.db`) |
